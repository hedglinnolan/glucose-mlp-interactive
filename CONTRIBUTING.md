# Contributing Guidelines

## Code Style

- Follow PEP 8 for Python code
- Use type hints where appropriate
- Add docstrings to all functions and classes
- Keep functions focused and small (< 50 lines when possible)

## Error Handling

- Always use try-except blocks for file I/O and external API calls
- Provide meaningful error messages to users
- Log errors with appropriate levels (INFO, WARNING, ERROR)
- Never expose sensitive information in error messages

## Testing

- Test with various dataset sizes and formats
- Test edge cases (missing data, single column, etc.)
- Verify all models train successfully
- Check visualizations render correctly

## Documentation

- Update README.md for new features
- Add docstrings to new functions
- Update QUICKSTART.md if workflow changes
- Keep code comments concise and meaningful

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Update documentation
6. Submit pull request with clear description

## Questions?

Open an issue for questions or suggestions.
